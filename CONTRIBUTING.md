# Contributing

Thanks for your interest in contributing.

## Guidelines

- **Open an issue first** -- Before starting work on a feature or fix, open an issue to discuss the approach.
- **Keep PRs focused** -- One concern per pull request. Small, reviewable changes merge faster.
- **No code snippets in docs** -- Documentation should describe commands and concepts, not embed source code.
- **Test locally** -- Make sure the bridge server starts, the extension loads, and prompts round-trip before submitting.

## Getting Started

1. Fork the repo and clone your fork.
2. Create a virtual environment and install dependencies from `requirements.txt`.
3. Load the extension in developer mode and verify the WebSocket connection.
4. Create a branch, make your changes, and open a PR against `main`.
